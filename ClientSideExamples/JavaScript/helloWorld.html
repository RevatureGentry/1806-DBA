There are 6 primitive data types in JavaScript. They are

  1. string
  2. boolean
  3. number (64-bit floating point)
  4. null
  5. undefined
  6. Symbol (new in ES6)

Arrow Functions

Falsy refers to values that are not the boolean false, but evaluate to false. Values that are falsy include:

  • 0, The Number Zero
  • "", The Empty String
  • null
  • undefined
  • NaN

Note: null and undefined do not have == equality with any value except each other. NaN is not == or === to anything, including itself

Truthy refers to values that are not the boolean true, but evaluate to true. This is every other value except those listed above

There are 3 variables scopes in JavaScript

  1. Global
  2. Local
  3. Block - defined with the keywords let and const (introduced in ES6)

Functions in JavaScript differ from functions in other programming languages in the following ways

  1. Functions inherit from Object
  2. Functions can be anonymous. By this, we mean that functions can be assigned to variables
  3. Functions can be self-invoking. This is achieved by wrapping our function in a set of parentheses and having a trailing set of parentheses for any arguments
  4. Functions can be hoisted. By this, we mean that functions can be invoked in a script before their implementation
  5. Arrow functions were introduced in ES6 and they help provide a more Object-Oriented feel to JavaScript. Every time we create a declared function in JS, it creates a new value for this. If we use an arrow function, it inherits the this value from its enclosing context.

Fundamentally, Objects are just a collection of key-value pairs

Objects can be created in the following ways

  1. Through Object Literal Notation (using curly braces)
  2. By instantiating a new Object()
  3. With a constructor

Events in JavaScript are reactions to user actions. In general, they begin with on ("onclick", "ondblclick", "onload", etc)

Event Listeners are attached to HTML elements and wait for its specified event to occur, and invokes its specified callback function when the event occurs

We can get references to HTML elements in 5 (main) ways

  1. document.getElementById => returns a reference to the HTML element with the specified ID
  2. document.getElementsByClassName => returns an array of HTML elements that contain the specified class name
  3. document.getElementsByTagName => returns an array of HTML elements of the specified tag name
  4. document.querySelector => returns the first instance of an HTML element with the specified CSS selector
  5. document.querySelectorAll => returns an array of HTML elements matching the specified CSS Selector

Events are executed in one of two ways

  • Bubbling is the execution of events from the innermost tags to the outermost tags
  • Capturing is the execution of events from the outermost tags to the innermost
